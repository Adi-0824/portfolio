Title: Hospital Management System: A Web-Based Enterprise Platform

The thesis reports the analysis, design, implementation, and clinical evaluation of a layered enterprise platform designed for local medical centers. It replaces fragmented paper files with unified persistence schemas mapped via JPA Hibernate to an H2 database.

Dissertation Chapters Outline

Chapter 1: Objectives & Scope of the Project
Goal is to deliver integrated operations, automatic invoicing rules, persistent local H2 storage, and a premium GUI. Scaled for localized multi-department clinics to decrease paper logs and clerical overhead.
Chapter 2: Theoretical Background
Review of structural web architectures (Controller-Service-Repository), Spring Boot DI, JPA Hibernate ORM, and Single Page Application (SPA) clients fetching asynchronous REST payloads.
Chapter 3: Definition of the Problem
Outlines clinic challenges in Raipur: paper register transcription delays, double-booking doctor slots, revenue leaks from unbilled completed doctor consultations, and lack of real-time administrative counters.
Chapter 4: System Analysis & Design
Establishes 10 functional and 5 non-functional requirements. Outlines the entity database mappings (Doctor, Patient, Appointment, Billing relationships) and Data Flow Diagrams (Level 0 and Level 1).
Chapter 6: Methodology & Implementation Details
Details the adoption of the Incremental Lifecycle Model. Discusses backend configurations, Maven XML properties, and controller endpoints. Features the business rules connecting completed appointments with H2 transactions.
Chapter 11: Testing Methodology & Test Report
Logs JUnit test executions checking Spring Boot contexts and database seeder validations. Documents manual verification for light/dark CSS transitions, REST request parameters, and average API load times under 50ms.
